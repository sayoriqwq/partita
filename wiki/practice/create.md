# Create

Create a skill by following this order:

1. Capture or read a real case.
2. Read pressure from the case.
3. Write the assertion.
4. Decide primitive or orchestrator.
5. Define boundary, effects, duration, and validation.
6. Project to Codex files using [[projection/codex/skill-md]] V1 shape.
7. Run generation and verification.

See [[skill/assertion]] and [[projection/codex/skill-md]].
