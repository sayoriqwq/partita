# Density 符号

本 reference 定义 `density` 激活后的稳定符号词表。

## 原则

- 符号只在能唯一表达关系时使用。
- 当符号会丢失语用 nuance 时，MUST 使用受控中文。
- 不为已经更短、更清楚的中文关系发明新 operator。

## 词表

| Symbol | Meaning |
| --- | --- |
| `A -> B` | 原因、推导或流向 |
| `A => B` | 决策后果或执行结果 |
| `A = B` | 定义或等价 |
| `A ≠ B` | 排除或不等价 |
| `A > B` | 优先级 |
| `A/B/C` | 候选项，不表示同时成立 |
| `A & B` | A 与 B 同时成立 |
| `A: B` | 字段、标签或 key-value 关系 |
| `!A` | 禁止 A |
| `A -x> B` | A 阻断 B |
| `A @B` | A 归属或挂载在 B |
| `A @>B` | A 应用于 B |
| `[state]` | 当前状态 |
| `(?)` | 不确定或需要确认 |

`A @>B` 不继承 `>` 的优先级语义。

## 验证标记

- `✅` 表示正例或合规 case。
- `❌` 表示反例或不合规 case。
- 只在 examples、review 或 validation 中使用。
- 不把 `✅` 和 `❌` 当作 inference chain 中的 semantic operator。

## 禁用形式

- 使用 `A -x> B`，不要使用 `A !-> B` 表示阻断。
- 使用受控中文表达递进连接，不要使用 `A &> B`。
- 使用 `A @>B`，不要使用 `A @=>B` 表示适用关系。
