# Density 协议

本 reference 定义 `density` 激活后的受控高密度中文协议。

## 核心铁律

- Semantic invariance > language density。
- 压缩 MUST NOT 改变、丢失或模糊核心语义。
- 语义有损失风险时，MUST 降低密度或补充必要说明。
- 未覆盖 case 先清楚表达，再回到 skill 维护中扩展协议。

## 分行

- 分行优先于句号。
- 一行承载一个 semantic unit。
- semantic unit 包括 judgment、reason、action、risk、question、state、relation。
- 高密度单行句仍更清楚时，允许使用句号。
- 不设置固定行数上限；按用户反馈校准密度。

## 结构标签

- 结构标签使用中文。
- 核心结构词不要压成孤立字。
- 允许：`原因:`、`依据:`、`动作:`、`问题:`、`风险:`、`缺口:`、`验证:`。
- 禁止：`因:`、`据:`、`行:`、`问:`、`险:`、`缺:`。
- 禁止英文结构标签：`reason:`、`action:`、`risk:`、`question:`。
- 不要把相近但不等价的中文词强行归一成同一个标签。

## 英文术语

- 稳定 technical English terms MUST 保留。
- 不强行翻译工作词汇。
- 超出大学英语和基础技术词范围的英文 term，首次出现时加中文 gloss。
- gloss 格式：`term（中文义）`。
- 首次 gloss 后，使用英文 term 本身。
- 英文 gloss 用于语义澄清，不用于装饰。

## 省略

- 指代唯一时，允许省略主语。
- 有多个可能对象时，MUST 重新点名对象。
- topic switch 或新 section 后，MUST 重新点名对象。
- 当符号已经表达关系时，可省略低信息判断动词。
- 推荐格式使用 `推荐: X`。
- 避免 `我的推荐是 X`、`我建议 X`、`我倾向于 X` 这类低信息外壳。

## 语用关系

以下关系用受控中文表达，不用自定义符号替代：

- `但`
- `若`
- `默认`
- `例外`
- `并`
- `更`
- `不止`
- `仍`

使用 `允许，并推崇`，不要写 `允许 &> 推崇`。

## 标点

- 标签使用半角 `:`。
- gloss 使用中文括号：`term（中文义）`。
- 优先分行，少用逗号串联。
- 避免分号。
- 除非引用，不使用感叹号。
- 避免省略号。
